## Konfiguracja w kernelu 

    devtool modify linux-stm32mp
    devtool menuconfig linux-stm32mp

Szukamy BMP280 (z pomocą `/`)

Zaznaczamy że ma się kompilować `*`

Zapisujemy i wychodzimy z menuconfiga

Zapisujemy:

    devtool finish linux-stm32mp ../layers/meta-ifm

Sprawdzamy czy mam nowy fragment konfiguracyjny

    layers/meta-ifm/recipes-kernel/linux/linux-stm32mp/devtool-fragment.cfg

Do recepty z devtoola dodaj linijkę:

    KERNEL_CONFIG_FRAGMENTS:append = " ${WORKDIR}/devtool-fragment.cfg"

## Test

W subsystemie IIO powinno pojawić się nowe urządzenie (domyślnie są dwa)

    root@stm32mp25-disco-e3-d2-a5:~# ls /sys/bus/iio/devices/                                  
    iio:device0  iio:device1  iio:device2

Czujnik powinien się przedstawić:

    root@stm32mp25-disco-e3-d2-a5:~# cat /sys/bus/iio/devices/iio:device0/name                 
    bme280

I pozwolić na odczyt temperatury:

    root@stm32mp25-disco-e3-d2-a5:~# cat /sys/bus/iio/devices/iio:device0/in_temp_input        
    27330

## Debugowanie

If the BME280 sensor is not visible in userspace (usually in `/sys/bus/iio/devices/`), you should follow this systematic debugging path from the hardware up to the driver.

### 1. Sprawdź szynę i2c
Sprawdzamy czy odpowiednia magistrala jest w systemie:

```bash
ls /dev/i2c-*
```

Dodatkowo możesz uzyć 

    i2cdetect -l

 Do identyfikacji konkretnej magistrali. 

### 2. Skan i2c
Nawet jeżeli sterownik nie zadziała, sprzęt powinien odpowiedzieć prawidłowo na skan.


```bash
i2cdetect -y 1  # Zamień 1 na numer magistrali
```
*   **Result:** You should see `76` (or `UU` if the driver is already bound to it) at the intersection.
*   **Jeżeli nic nie widać:** Sprawdź połączenia (SDA/SCL) i zasilanie (3.3V). Zamień SDA/SCL jeśli nie jesteś pewien.

### 3. Sprawdź logi kernela
If the hardware is detected but the driver didn't create the IIO device, the kernel likely tried to "probe" it and failed.

```bash
dmesg | grep -iE "bmp280|bme280|i2c"
```


### 4. Konfiguracja sterownika

Sprawdź czy driver został poprawnie załadowany.

```bash
# Czy został załadowany moduł
lsmod | grep bmp280

# Sprawdzenie configa kernela
zcat /proc/config.gz | grep CONFIG_BMP280
```

### 5. Sprawdź devicetree
Sprawdź czy system widzi odpowiedni węzeł w proc/


```bash
find /sys/firmware/devicetree/base/ -name "*bme280*"
```
*   **Jeżeli coś jest:** Sprawdź status: `cat /sys/firmware/devicetree/base/.../bme280@76/status`. Powinien być `okay`.
*   **Jeżeli nic nie ma** Overlay nie został nałożony. Sprawdź local.conf i ustawienia w bitbake -e
